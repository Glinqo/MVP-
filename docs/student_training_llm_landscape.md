# Student Training LLM Landscape

调研日期：2026-07-08

## 调研结论

现有面向学生训练的大模型产品，大多集中在五类能力：

1. 题目答疑与分步讲解。
2. 苏格拉底式追问和学习引导。
3. 练习题、测试卷、错题本和学习报告。
4. 语言学习中的口语角色扮演、作文批改和翻译。
5. 自适应学习路径、学情画像和教师辅助。

它们的优势是覆盖面广、交互自然、能降低一对一辅导成本；主要不足是容易停留在“题目/知识点训练”，对职业教育实训中的设备安全、接线证据、故障链路、岗位能力和教师可审核评分支持不足。

因此，我们产品的差异化不应是再做一个“AI 答疑老师”，而应是：

**面向机电实训现场的安全排故导航 + 岗位能力诊断 + 确定性评分 + 实训任务推荐。**

## 已有功能模块总结

### 1. AI 答疑与分步讲解

典型功能：

- 拍照或输入题目。
- 识别知识点。
- 分步骤讲解。
- 提供不同解法。
- 追问学生理解情况。

优点：

- 对学生即时帮助明显。
- 能缓解教师一对一答疑压力。
- 容易形成可演示产品。

缺点：

- 容易变成“直接给答案”。
- 学生可能看懂过程但不会独立完成。
- 对操作类实训，文字解释不能替代现场证据检查。

### 2. 苏格拉底式学习引导

典型功能：

- 不直接给答案。
- 通过问题提示学生回忆前置知识。
- 按学生回答继续追问。
- 鼓励自我解释和反思。

优点：

- 比普通问答更符合学习规律。
- 能减少“抄答案”。
- 对概念理解和数学推理有帮助。

缺点：

- 如果学生动机不足，可能觉得慢。
- 对实训故障，光追问不够，还需要明确检查顺序。
- 没有规则约束时，仍可能追问偏题。

### 3. 自适应测评与学习路径

典型功能：

- 诊断题。
- 错题本。
- 薄弱知识点定位。
- 个性化学习路径。
- 学习报告。

优点：

- 能实现“千人千面”。
- 教师可查看学情。
- 和我们的诊断/推荐闭环高度相关。

缺点：

- 多数系统按“知识点”推荐，不按“岗位任务”推荐。
- 推荐资源常停留在视频、题目和讲解。
- 职业教育需要把薄弱点落到实训任务、设备检查和评价标准。

### 4. 语言训练与写作批改

典型功能：

- 口语角色扮演。
- 发音、语法、表达反馈。
- 作文批改和润色。
- 翻译和词典增强。

优点：

- 多轮对话天然适合语言学习。
- 反馈即时，使用频次高。
- 场景相对安全。

缺点：

- 与机电实训距离较远。
- 主要评价语言表达，不评价操作过程和故障证据。

### 5. 教师辅助与课堂反馈

典型功能：

- 备课。
- 组卷。
- 批改。
- 学情分析。
- 教学建议。

优点：

- 能减轻教师重复劳动。
- 容易形成教师端价值。
- 有助于课程改进。

缺点：

- 很多产品教师端和学生端数据链不够细。
- 对职业实训而言，教师需要的是“学生错在哪个操作环节”，不是只看题目分数。

## 共同优点

- **即时性**：学生随时可以问、练、测。
- **个性化**：可根据学生水平调整解释和练习。
- **低成本扩展**：比真人一对一更容易规模化。
- **多模态**：拍照、语音、文本、PDF 都能进入学习流程。
- **教师减负**：组卷、批改、答疑、学情报告可自动化一部分。

## 共同缺点

- **容易替代思考**：如果直接给答案，学生练习表现会变好，但独立考试或实操能力未必提升。
- **缺少现场证据链**：很多系统知道题目答案，却不知道设备现场状态。
- **评分可信度问题**：LLM 自由评分难以审计，职业技能评价尤其不能只靠模型判断。
- **安全边界不足**：通用 AI Tutor 很少内置接线、通电、气动执行机构等安全约束。
- **职业岗位映射弱**：多数产品围绕学科知识点，不围绕岗位能力、工作任务、实训评价。
- **教师审核入口不足**：生成内容若直接给学生，可能不符合校内设备、教材或实训规范。

## 对我们产品的启发

### 应该借鉴

- 借鉴 ChatGPT Study Mode 和 Khanmigo 的“引导式学习”，避免直接给结论。
- 借鉴自适应学习产品的“错题 -> 薄弱点 -> 路径”闭环。
- 借鉴 RAG/学习工具的来源引用和学习报告。
- 借鉴语言学习产品的场景化对话，把学生带入真实故障情境。

### 应该避开

- 不做泛学科答疑。
- 不做拍照搜题式答案机。
- 不让 LLM 自由判分。
- 不推荐脱离设备和安全规范的操作。
- 不把学习路径只做成视频/知识点列表。

### 我们的切入点

我们可以把“学生大模型培训”的通用能力，改造成职业教育实训能力：

```text
通用 AI Tutor:
题目 -> 讲解 -> 练习 -> 错题 -> 学习路径

我们的产品:
故障现象 -> 安全检查 -> 状态追问 -> 能力图谱 -> 规则评分 -> 薄弱能力 -> 实训任务
```

最能形成差异化的功能：

1. 三联状态对照：传感器灯、PLC 输入灯、在线监控。
2. 安全优先排故：接线/通电/气动调试先过安全节点。
3. 岗位能力图谱：把错题映射到岗位能力，而不是只映射到知识点。
4. 确定性评分：评分来自规则和题库，不由 LLM 自由判断。
5. 实训任务推荐：薄弱点最终落到一节课可做的训练任务。
