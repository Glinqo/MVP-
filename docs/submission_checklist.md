# Submission Checklist

## 必交材料

- [ ] README 能说明项目目标与运行方式。
- [ ] 本地 Demo 运行截图或操作说明已准备。
- [ ] 可选平台工作流截图或导入说明已准备（如需要）。
- [ ] `prompts/` 中每个节点提示词可被本地服务或平台节点复用。
- [ ] `knowledge/` 中有能力节点、知识点、资源和训练任务。
- [ ] `diagnosis/` 中有诊断题、评分规则和示例答案。
- [ ] 评分代码已在本地测试或 API 调用中验证。
- [ ] `tests/scoring.test.js` 可通过。
- [ ] 演示脚本能在 5 分钟内讲完。

## 演示前自查

- [ ] 输入一个学生问题，能进入正确意图。
- [ ] 输入一组诊断答案，能得到总分、节点分和错误标签。
- [ ] 至少一个薄弱节点能生成学习路径。
- [ ] 教师视图能输出班级共性问题。
- [ ] 所有 JSON 文件格式有效。

## 风险提示

- 不要承诺真实控制设备。
- 不要声称替代教师评价。
- 不要把模型输出当作唯一评分依据。
- 演示时强调本项目是可扩展 MVP。
