## 新增文件

### `app/services/intent.py` — 意图分类器

实现了 LLM 优先、关键词兜底的双路意图分类器。LLM 路径调用 `prompts/intent_classifier_prompt.md` 做零温度分类，返回意图标签、置信度和原因；关键词路径使用加权匹配（强信号如"帮我出"权重为 2，弱信号如"PLC"权重为 0.3），diagnosis 在平局时主动让位以减少技术通用词误触发。当输入完全不含任何技术关键词时默认归类为 clarify。支持 6 种意图：diagnosis（故障排查）、knowledge_qa（知识问答）、quiz（出题自测）、graph（能力图谱查询）、learning_path（学习路径推荐）、clarify（信息不足需追问）。

### `app/services/intent_handlers.py` — 意图处理器

为每种非 diagnosis 的意图创建了独立的处理器函数，均返回与原有 `chat_message()` 保持一致结构的响应字典，前端无需改动即可正确渲染。`handle_knowledge_qa` 调用知识检索后由 LLM 基于检索结果生成带来源引用的回答；`handle_quiz` 调用个性化出题服务生成练习题；`handle_graph` 调用图谱生成服务输出能力节点列表；`handle_learning_path` 调用培养方案服务输出训练单和阶段计划；`handle_clarify` 加载 `prompts/clarify_prompt.md` 通过 LLM 生成口语化的追问（带"为什么需要这些信息"的解释），同时提供了 3 轮追问限流机制，超限后自动给出基于已有信息的最佳诊断。所有处理器在 LLM 不可用时均自动回退到静态模板。

### `docs/模块开发日志/功能调用协议.md`

定义了 8 个后端模块的完整接口签名（输入参数类型、返回 JSON 结构、字段说明），包括意图分类器、规则引擎、知识检索、图谱生成、题目生成、培养方案生成、LLM 澄清追问、学生画像。同时记录了 Chat API 的 Request/Response 合约和前端工具调用的绑定关系。

### `docs/模块开发日志/对话状态结构.md`

定义了 10 种核心数据结构的形式化描述：Session（会话生命周期）、UserMessage/AssistantMessage（消息格式）、IntentResult（意图分类结果）、EvidenceItem（证据条目）、KnowledgeItem（知识引用）、AbilityItem（能力命中）、TaskCard（任务推荐）、LearnerContext（学生能力画像）、Event（学习事件）、前端 State（app.js 全局状态）。每种结构包含字段说明、示例 JSON 和前后端对应关系。

## 修改文件

### `app/services/chat.py`

主要改动集中在 `chat_message()` 函数入口处。在原有 `assist()` 调用之前新增了意图分类步骤：先调用 `classify_intent()` 获取意图标签，然后根据意图分流到对应处理器（quiz/graph/learning_path/knowledge_qa/clarify 直接 return），diagnosis 意图继续走原有的 assist 流程保持不变。同时在 diagnosis 路径的 `need_clarification` 分支中新增了 LLM 驱动的澄清追问逻辑：当规则引擎判定信息不足时，优先用 `generate_clarify_questions()` 生成自然追问，替代原有的 JSON 静态追问。整个过程中 diagnosis 路径原有逻辑零改动、零回归。新增了 `_finalize()` 辅助函数，为非 diagnosis 意图提供统一的 session 事件记录和学生图谱更新。

### `web/app.js`

重构了对话消息的附件渲染逻辑。原有的 `renderMessageExtras()` 使用单个可折叠的 `<details>` 面板笼统展示所有证据和引用，改为 `renderMessageCards()` 拆分为 5 种独立的 inline 卡片组件在对话流中直接渲染。`renderEvidenceStrip()` 将证据渲染为圆角彩色标签条（橙=匹配现象、蓝=能力、绿=知识、紫=上下文）。`renderReasoningBar()` 将推理步骤渲染为编号步骤箭头链。`renderKnowledgeCards()` 将知识引用渲染为蓝色可展开卡片，卡片头显示 ID、主题和来源，点击可展开完整内容，底部带"追问"按钮。`renderAbilityCards()` 将能力命中渲染为黄色卡片，显示能力名、命中标签和原因描述，底部带"问 AI 讲解"按钮。`renderTaskCards()` 将任务推荐渲染为绿色卡片，显示任务类型标签、标题、预估时长和交付物描述，底部带"开始这个任务"按钮。所有卡片的交互按钮通过已有的 `attachAskButtons()` 统一绑定到讲解抽屉。在 `renderMessages()` 末尾增加了 `attachAskButtons()` 调用，确保每次消息刷新后新卡片按钮的事件都能正确注册。

### `web/styles.css`

新增了约 150 行卡片样式，定义了 6 个核心 CSS 类。`.chat-cards` 作为卡片容器提供统一间距和顶部分割线。`.kb-card-inline` 定义知识卡片的蓝色边框、悬停加深效果、ID/主题/来源三栏头部布局、内容两行截断与展开切换、标签组和操作按钮区。`.ability-card-inline` 定义能力命中卡片的黄色边框、能力名加粗、命中/薄弱两色标签、原因描述和操作按钮区。`.task-card-inline` 定义任务卡片的绿色边框、类型标签（实训/学习资料）、元信息行和操作按钮区。`.evidence-strip` 定义证据标签条的 flexbox 布局和四种颜色圆点样式。`.reasoning-bar` 定义推理步骤的灰色箭头连接样式。所有卡片均使用项目的 CSS 变量体系（颜色、圆角、阴影），与现有 UI 一致。
