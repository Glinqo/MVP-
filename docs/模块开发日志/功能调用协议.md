# 功能调用协议

版本：0.2.0
日期：2026-07-11

定义对话模块（模块一）与后端各功能模块之间的调用协议，包括意图路由表、模块接口签名、Chat API 合约。

---

## 一、意图路由表

`classify_intent()` 输出 → `chat_message()` 根据 intent 字段路由到对应处理器：

| intent | 路由目标 | 调用模块 | 说明 |
|---|---|---|---|
| `diagnosis` | 原 assist 流程 | `assist()` → `chat_completion()` | 故障诊断 + 能力命中 + 知识缺口 |
| `knowledge_qa` | `handle_knowledge_qa()` | `search_knowledge()` → `chat_completion()` | RAG 检索 + LLM 回答 |
| `quiz` | `handle_quiz()` | `personalized_quiz()` | 个性化练习题生成 |
| `graph` | `handle_graph()` | `build_ability_graph()` | 岗位能力图谱 |
| `learning_path` | `handle_learning_path()` | `personalized_plan()` | 个人培养方案 |
| `clarify` | `handle_clarify()` | `generate_clarify_questions()` → `chat_completion()` | LLM 自然追问（3 轮限流） |

## 二、模块接口签名

```
classify_intent(message: str, history?: list, context?: dict) -> IntentResult
```

**IntentResult**:
```json
{
  "intent": "diagnosis | knowledge_qa | quiz | graph | learning_path | clarify",

  "reason": "关键词匹配 → diagnosis",
```

## 四、前端工具调用绑定

`web/app.js` 中 `TOOLS` 常量与后端模块的对应关系：

| tool_id | 后端模块 | 触发方式 |
|---|---|---|
| `dashboard` | `build_student_dashboard()` | 用户点击按钮 / 对话自动刷新 |
| `job_graph` | `build_job_ability_graph()` | 用户点击按钮 |
| `student_graph` | `build_student_ability_graph()` | 用户点击 / 对话后自动更新 |
| `graph` | `build_ability_graph()` | 用户点击 / 诊断对话后自动生成 |
| `knowledge` | `search_knowledge()` | 对话中作为 knowledge_refs 展示 |
| `tasks` | remediation_cards | 对话中作为 task-card-inline 展示 |
| `quiz` | `personalized_quiz()` | 用户点击 / quiz intent 路由 |
| `scenario` | `start_scenario() / step_scenario()` | 用户点击 |
| `plan` | `personalized_plan()` | 用户点击 / learning_path intent 路由 |
| `teacher` | `teacher_summary()` | 用户点击 |
