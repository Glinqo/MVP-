# 对话状态结构

版本：0.2.0
日期：2026-07-11

定义模块一（对话与产品流程）涉及的所有核心数据结构，前后端统一。

---

## 一、整体架构

```
Session (会话)
  ├─ Messages[] (消息列表)
  │    ├─ UserMessage / AssistantMessage
  │    └─ MessageMeta (证据/推理/卡片数据)
  ├─ IntentResult (当前消息的意图分类)
  └─ LearnerContext (学生能力画像)

跨 Session 共享：
  └─ JobProfile (岗位画像，全局单例)
```

---

## 二、Session（会话）

```json
{
  "session_id": "demo-1720000000000",
  "created_at": "2026-07-11T10:00:00Z",
  "updated_at": "2026-07-11T10:05:00Z",
  "events": [Event],
  "feedback": "已掌握 | 仍不会 | 需要更基础讲解",
  "user_input": "最近一次用户输入",
  "score_result": {},
  "weak_abilities": [],
  "recommended_path": []
}
```

**生命周期**：
- 创建：`chat_start()` 或首次 `chat_message()` 时自动生成
- 更新：每次 `chat_message()` 调用 `append_session_event()` 追加事件
- 持久化：`data/sessions/{session_id}.json`

## 四、IntentResult（意图分类）

```json
{
  "intent": "diagnosis | knowledge_qa | quiz | graph | learning_path | clarify",
  "confidence": 0.85,
  "reason": "LLM 分类: 学生描述故障现象",
  "slots": {
    "requires_safety_notice": true,
    "mentioned_signals": ["传感器灯", "PLC输入"]
  },
  "source": "llm | keyword"
}
```

**分类优先级**：LLM > 加权关键词 > 默认 clarify

## 五、EvidenceItem（证据）

```json
  "label": "匹配现象 | 命中能力: XXX | 知识引用: K011 | 传感器动作灯",
```

```json
  "session_id": "demo-123",
  "summary": "已记录 5 条学习证据；薄弱：传感器接线判断、PLC I/O 映射...",
  "event_count": 5,
  "weak_abilities": [

      "ability_name": "传感器接线判断",

      "confidence": 0.7,
      "evidence_count": 3,
      "update_reasons": ["学生2次提交答案错误", "学生自评仍不会"],
      "next_best_action": "完成传感器接线判断训练任务",
      "source": "student_graph"
    }
  ],
  "improving_abilities": [...],
  "recommended_next": [...],
  "touched_abilities": [...],
  "mastered_abilities": [...],
  "recent_events": [...],
  "next_best_actions": [...]
}
```

**状态流转**：
```
unknown → touched (首次命中) → weak | improving (评估后)
  → improving (学习/训练后) → mastered (掌握确认)
```
