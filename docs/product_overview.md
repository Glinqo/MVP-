# 产品功能与亮点说明

## 一句话定位

这是一个面向高职机电一体化职业新人的轻量岗位培训 Agent。它不把学生先拉去刷题，而是让学生像在实训现场一样直接提问：例如“传感器灯亮但 PLC 没输入，为什么？”系统先给安全提醒，再追问关键现场证据，随后直接给出排故思路，并把问题暴露出的能力缺口同步到能力图谱、知识卡片和训练计划中。

当前默认岗位是：自动化生产线装调与运维技术员。

## 适用场景

- 高职机电一体化实训课：传感器 NPN/PNP 接线、PLC 输入公共端、I/O 地址映射、在线监控、气缸不动作排查。
- 职业新人岗前训练：围绕岗位任务学习，而不是围绕传统章节刷题。
- 教师或企业师傅带教：快速看到学生问答、自测、讲题和训练任务背后的薄弱能力。
- 竞赛演示：展示“问答辅导 -> 能力图谱 -> 知识缺口 -> 补救训练 -> 反馈闭环”的完整链路。

## 学生端主流程

1. 学生进入后确认岗位身份：自动化生产线装调与运维技术员 / 职业新人。
2. 学生在聊天主屏自由提问，也可以连续追问不懂的地方。
3. Agent 识别是否涉及接线、通电、PLC 监控、设备动作或气动调试；涉及时先输出安全提醒。
4. Agent 直接回答问题；如果证据不足，最多追问 3 个关键现场证据。
5. 系统把本次问题映射到能力节点、知识点、常见错误和实训任务。
6. 学生可打开全屏工具：岗位能力图谱、个人能力图谱、当前问题图谱、知识库、自测、实训任务、个人培养方案、教师摘要。
7. 学生做自测、看讲题、完成任务或提交反馈后，个人能力图谱继续更新。

## 核心功能

### 1. 问答式岗位培训主界面

- 第一入口是聊天，不是测试卷。
- 支持学生自由描述现场问题、追问概念、询问先学什么。
- 无大模型配置时使用规则兜底，有 OpenAI-compatible 配置时调用大模型增强解释。
- 回答中包含安全提醒、直接判断、可能原因、检查顺序、下一步追问建议和可用工具入口。

### 2. 问题驱动排故辅导

系统已内置典型问题模式：

- 传感器灯亮，PLC 输入灯不亮，在线监控不变。
- 传感器灯亮，PLC 输入灯亮，但程序或在线监控不对。
- 传感器灯不亮或不稳定。
- 气缸不动作，需要分电气、程序、气路排查。

每类问题都会关联必要追问、可能原因、能力节点、知识点、训练任务和来源字段。

### 3. 双能力图谱

岗位能力图谱：

- 展示岗位所需能力链路。
- 保留企业、行业、课程标准或教师整理来源。
- 支持岗位能力更新建议和教师确认机制。

学生个人能力图谱：

- 根据聊天命中、诊断错题、讲题记录、反馈和任务完成情况更新。
- 每个节点包含状态、掌握分、置信度、证据数量、更新时间和更新原因。
- 节点可打开证据详情，看到为什么被判定为薄弱、待确认、提升中或推荐下一步。

当前问题图谱：

- 高亮本次问题暴露的能力缺口。
- 同步展示关联知识卡片和推荐补救任务。

### 4. 知识缺口卡片

- 当前知识库聚焦电气安全、24V DC、三线制传感器、NPN/PNP、PLC 输入公共端、I/O 地址、常开/常闭、PLC 监控、输入点无响应、气缸不动作分层排查。
- 知识点保留 `source` 字段，支持教师后续核验和补充。
- 聊天、图谱、讲题和培养方案都优先引用知识库内容。

### 5. 确定性诊断与讲题

- 诊断题作为辅助验证功能，不再作为第一入口。
- 题目评分由 `diagnosis/scoring_rules.json` 和代码模块确定性完成。
- LLM 可以解释错因，但不能自由评分或覆盖规则结果。
- 学生可以在半屏讲题抽屉里看错因、标准答案、关联能力和补救任务。

### 6. 个性化培养方案

培养方案不是凭空生成，而是组合以下数据：

- 学生个人能力图谱中的薄弱点。
- 岗位能力图谱中的岗位要求和权重。
- 知识库文本讲解。
- `resources.json` 中的真实资源或视频链接。
- `training_tasks.json` 中的实训任务。
- 诊断题和检查点。

输出包括阶段计划、今日训练单、7 天补强计划、知识点讲解、资源链接、实训任务和复测检查点。

### 7. 实训角色扮演

- 内置排故场景，学生按现场证据一步步判断。
- 答对会记录提升证据，答错会记录薄弱证据。
- 场景训练会反向更新个人能力图谱。

### 8. 教师/师傅摘要

- 汇总学生薄弱能力、常见错误、推荐任务和下一次带教建议。
- 方便教师从“这名学生哪里不会”直接过渡到“下一节课怎么带”。

## 产品亮点

### 亮点一：从“做题诊断”改成“问题驱动学习”

传统系统通常让学生先做题，再给分数。本项目先回答学生眼前的问题，再诊断这个问题背后的能力缺口。这样更贴近实训现场，也更适合职业新人。

### 亮点二：能力图谱不是静态展示，而是证据系统

个人能力图谱由事件驱动更新：问答、自测、讲题、反馈、实训任务都能成为证据。每个节点都能解释“为什么这样判断”，避免黑盒画像。

### 亮点三：岗位图谱和个人图谱分开

岗位图谱回答“企业和行业需要什么能力”，个人图谱回答“学生现在掌握到了哪里”。两个图谱对比后，系统才能生成真正面向岗位差距的培养方案。

### 亮点四：专业安全约束内置到回答链路

涉及接线、通电、PLC 监控、传感器调试、设备动作或气动执行机构时，系统必须先给安全提醒，不提供绕过安全回路或冒险操作的建议。

### 亮点五：规则确定性与大模型解释分工明确

评分、薄弱点映射和图谱更新优先使用确定性规则；大模型只负责解释、追问引导和表达增强。这样既能演示 AI 辅导体验，又能保证评分可审计。

### 亮点六：借鉴优秀项目，但保持轻量自研

本项目借鉴了知识图谱问答、RAG 检索、学习者画像、Mastery Path、多 Agent 分工和工业 Graph-RAG 等开源项目思路，但没有引入重型数据库、账号系统或多 Agent 运行时。对应来源记录在：

- `docs/borrowed_feature_source_map.md`
- `docs/borrowed_feature_inventory.json`

## 数据与资产规模

- 能力节点：20 个以上，聚焦自动化生产线装调与运维技术员。
- 知识点：66 条，覆盖 NPN/PNP 传感器与 PLC 输入排故主线。
- 诊断题：20 道，支持单选、多选、排序和确定性评分。
- 训练任务：覆盖接线判断、公共端判断、I/O 映射、PLC 监控、气缸排查等。
- 问题模式：覆盖传感器灯亮/不亮、PLC 输入灯变化、在线监控异常、气缸不动作等典型现象。
- 资源库：包含文本资源、视频资源和课程/标准类来源字段。

## 技术实现

- 后端：Python 标准库 HTTP 服务，不依赖大型框架。
- 前端：静态 HTML/CSS/JS，聊天主屏 + 全屏工具工作台。
- 数据：本地 JSON/Markdown，不使用数据库。
- 评分：JavaScript 与 Python 双版本确定性模块。
- LLM：可选 OpenAI-compatible Chat Completions；无 API Key 时自动降级到规则回答。
- 图谱：第一版使用轻量 SVG/HTML/CSS 和 Mermaid 文本，不引入重型图谱库。

## 主要接口

- `POST /api/chat/start`：初始化岗位欢迎语和推荐提问。
- `POST /api/chat/message`：学生自由提问与追问。
- `POST /api/assist`：规则式问题排故分析兜底。
- `GET /api/graph/job`：岗位能力图谱。
- `GET /api/graph/student?session_id=...`：学生个人能力图谱。
- `GET /api/graph`：当前问题图谱或兼容图谱。
- `POST /api/explain`：题目、能力、知识、任务或消息讲解。
- `POST /api/score`：确定性诊断评分。
- `POST /api/diagnose`：薄弱点诊断。
- `POST /api/plan/personalized`：个性化培养方案。
- `GET /api/scenarios`、`POST /api/scenario/start`、`POST /api/scenario/step`：实训场景训练。

## 本地演示路径

启动服务：

```powershell
python .\app\server.py --port 8765
```

浏览器打开：

```text
http://127.0.0.1:8765
```

建议演示问题：

```text
传感器动作灯亮，但 PLC 输入点和在线监控都没有变化，为什么？
```

然后依次展示：

1. Agent 安全提醒和直接排故回答。
2. 关键追问和有价值追问建议。
3. 当前问题图谱高亮能力缺口。
4. 学生个人能力图谱的证据更新。
5. 诊断题自测和讲题抽屉。
6. 个性化培养方案和今日训练单。
7. 教师/师傅摘要。

## 后续可增强方向

- 将岗位需求材料导入流程做得更顺滑，支持教师粘贴招聘 JD 或企业任务描述后生成图谱更新建议。
- 增强图谱可视化交互，例如节点筛选、证据时间线、岗位图谱与个人图谱对比视图。
- 扩展真实设备排故场景，增加 PLC 输入模块、电磁阀、继电器、安全回路和气路压力类案例。
- 增加“讲完后复问”机制：学生看完讲解后用 1 道小题或一次现场判断验证是否真的懂了。
- 将资源库的视频与文本材料按能力节点做更细粒度匹配，避免泛泛推荐。
