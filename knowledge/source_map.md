# Source Map

本 MVP 的知识内容聚焦“传感器 NPN/PNP 接线与 PLC 输入信号排查”。当前专业来源仍处于待补充具体页码和文件编号阶段，不写成已经访谈确认。

## 文件关系

- `ability_nodes.json`：能力节点定义，是诊断和学习路径的主索引。
- `knowledge_50.md`：便于人工阅读的 50 个知识点。
- `knowledge_50.json`：便于工作流和代码读取的知识点结构化版本。
- `resources.json`：每个薄弱节点可推荐的资源。
- `training_tasks.json`：每个薄弱节点可推荐的实训任务。
- `common_errors.json`：诊断中的错误标签和补救提示。
- `imports/机电一体化智能体_知识库导入版_V1.md`：新导入的 64 条知识库人工阅读版，可作为星辰 RAG 知识库扩展源。
- `imports/机电一体化智能体_知识库导入版_V1.json`：新导入的 64 条知识库结构化版本，可作为星辰知识库导入候选。
- `../docs/references/知识库2.md`：来源依据、课程标准、教材、MOOC 和技术论坛参考索引。

## 已导入资料结论

- `机电一体化智能体_知识库导入版_V1.*` 与 MVP 场景匹配，覆盖岗位能力图谱、电气安全、接线图识读、NPN/PNP、PLC 输入监控、气缸不动作和反馈闭环。
- `知识库2.md` 可用于后续细化 `source` 字段，但其中来源仍需人工核验具体链接、页码或章节。
- `与竞赛相关的AI开源模型.docx` 只作为架构调研，不作为本 MVP 的运行依赖。

## Source 字段写法

当前允许写法：

- `PLC 实训指导书/教师确认资料/课程标准待补充`
- `机电一体化智能体_知识库导入版_V1`
- `知识库2.md 来源依据待核验`
- `project_curated`

禁止写法：

- 不写“已访谈确认”。
- 不写未核验的具体页码。
- 不把开源项目 README 当作专业教学标准。

## 扩展规则

新增知识点时，需要同时检查：

- 是否已有对应能力节点。
- 是否能映射到一个训练任务。
- 是否需要新增常见错误标签。
- 是否会影响评分规则。
- 是否需要同步更新星辰 Retrieve 节点导入说明。
